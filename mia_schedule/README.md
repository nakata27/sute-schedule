# MIA Schedule App 📅

Приложение для отображения расписания студентов КНУТЭ с сайта https://mia1.knute.edu.ua/time-table/group

## ✨ Возможности

- 📚 **Получение расписания** для любой студенческой группы
- 🎨 **Темная "люкс" тема** в стиле Apple/Telegram
- 🌍 **Двуязычность**: Украинский (по умолчанию) + Английский
- 📱 **Адаптивный дизайн** для всех устройств
- 💾 **Кэширование** для быстрой работы
- 🔄 **Переключение недель** - текущая, следующие недели
- 📊 **Цветовая индикация** типов занятий:
  - 🟡 Лекции - бледно-желтый
  - 🟢 Практики - бледно-зеленый  
  - 🔴 Сессии - бледно-красный

## 🏗️ Архитектура

```

└── tests/               # Тесты
```

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
cd mia_schedule
python -m venv venv
source venv/bin/activate.fish  # или source venv/bin/activate для bash
pip install -r requirements.txt
```

### 2. Тестирование Backend

```bash
python tests/test_backend.py
```

### 3. Запуск приложения

```bash
python app.py
```

Откройте в браузере: http://127.0.0.1:5000

## 📊 Структура данных

### Lesson (Занятие)
- `lesson_number` - Номер пары (1-7)
- `start_time` / `end_time` - Время начала/окончания
- `lesson_date` - Дата
- `subject` - Название предмета
- `lesson_type` - Тип (lecture/practice/session)
- `room` - Аудитория
- `teacher` - Преподаватель

### DaySchedule (День)
- `day_date` - Дата
- `day_of_week` - День недели (0-6)
- `lessons` - Список занятий

### WeekSchedule (Неделя)
- `week_number` - Номер недели
- `week_start` / `week_end` - Начало/конец недели
- `days` - Расписание по дням

## ⚙️ Конфигурация

Настройки в `config/settings.py`:

```python
# Кэширование
USE_CACHE = True
CACHE_LIFETIME_HOURS = 24

# Rate limiting
RATE_LIMIT_DELAY = 1.0  # секунды между запросами
MAX_RETRIES = 3

# Хранилище
USE_SQLITE = False  # False = JSON, True = SQLite
```

## 🎨 UI компоненты

1. **Выбор группы** (первый запуск)
   - Факультет → Курс → Группа
   - Сохранение выбора локально

2. **Главный экран**
   - Текущая неделя с датами
   - Список дней с занятиями
   - Цветные плитки для каждой пары
   - Кнопка "информация" для деталей

3. **Боковое меню**
   - Изменить группу
   - Показать расписание
   - Настройки (язык)
   - Контакты разработчика (внизу)

4. **Модальное окно** с деталями занятия
   - Полная информация о паре
   - Преподаватель
   - Аудитория
   - Комментарии

## 🔄 Workflow

1. **Первый запуск** → Выбор группы → Сохранение настроек
2. **Загрузка данных** → Проверка кэша → Запрос к сайту (если нужно)
3. **Отображение** → Текущая неделя → Переключение недель
4. **Обновление** → Pull-to-refresh или кнопка обновить
